## [0.3.3] - 2026-01-27

### 🚀 重大变更
- **工具分类系统** - 智能三层分类机制（精确匹配 → 规则引擎 → LLM fallback），自动路由 AI 请求到最合适的工具
- **Agent 服务模块化** - 重构 Agent 系统架构，提取工具注册表、事件监听器、去重器等核心服务模块
- **自定义提供商增强** - 修复 NVIDIA 等自定义提供商 404 错误，添加示例模型列表，优化用户配置体验

### ✨ 新增功能

#### 工具分类系统 (Tool Classification System)
- **三层智能分类**：
  - Layer 1 精确匹配：直接识别命令如 `/git status`、`/npm install`
  - Layer 2 规则引擎：基于关键词模式识别（搜索、代码生成、分析、终端等）
  - Layer 3 LLM fallback：复杂语义由 LLM 智能判断
- **工具注册表** - 统一管理所有内置工具（bash、agent_read_file、agent_write_file、agent_list_dir 等）
- **分类结果缓存** - 提升重复请求的性能，减少 LLM 调用
- **完整测试覆盖** - 70+ 单元测试覆盖所有分类规则

#### Agent 服务模块化 (Agent Service Modularization)
- **工具注册表 (ToolRegistry)** - 统一工具定义、注册、查找接口
- **事件监听器 (AgentListeners)** - 标准化 Agent 事件处理（进度、工具调用、完成、错误）
- **去重器 (ToolCallDeduplicator)** - 防止重复执行相同工具调用
- **格式化器** - Task Tree、Markdown、增量解析器等多种输出格式

#### 自定义提供商增强 (Custom Provider Enhancement)
- **字段命名修复** - 解决前端 camelCase 与后端 snake_case 不匹配问题
- **默认模型列表** - 自定义预设添加示例模型（z-ai/glm4.7、nv-tmp 等）
- **模型为空验证** - "设为默认"按钮在模型为空时禁用并提示用户

### 🐞 修复
- **自定义提供商 404 错误** - 修复 NVIDIA 等自定义提供商因字段映射导致的 API 调用失败
- **商业版字段映射** - 修复 ifainew-core 中的 base_url 和 modelName 回退逻辑
- **目录列表显示** - 过滤系统目录（如 `.git`、`node_modules`），优化目录列表可读性
- **文件统计功能** - agent_list_dir 添加文件和目录数量统计
- **文件内容显示** - agent_read_file 修复显示完整内容的问题

### 🧪 测试
- **工具分类基线测试** - 建立 70+ 测试用例的基线，防止回归
- **E2E 测试优化** - 添加 Tauri 模式检测，避免在非 Tauri 环境执行特定测试
- **回归测试标记** - 标识需要回归验证的 E2E 测试脚本

## 版本说明

### 版本号规则
- **主版本号 (Major)**: 重大架构变更或不兼容的 API 修改
- **次版本号 (Minor)**: 新增功能,保持向后兼容
- **修订号 (Patch)**: Bug 修复和小改进

### 变更类型标记
- `新增` / `Added` - 新功能
- `变更` / `Changed` - 现有功能的变更
- `弃用` / `Deprecated` - 即将移除的功能
- `移除` / `Removed` - 已移除的功能
- `修复` / `Fixed` - Bug 修复
- `安全` / `Security` - 安全性改进
